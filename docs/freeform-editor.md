# 自由编辑器

本文描述当前自由编辑器的 v3 场景树契约和用户可见行为。自由编辑器默认使用浏览器本地存储；配置 `VITE_API_BASE` 后使用远程账号、草稿和图片服务。两种存储模式共享同一份场景树和编辑器状态机。

## 文档模型

自由编辑文档固定使用 `documentVersion: 3`。每个页面有自己的尺寸、背景和递归 `nodes`，节点可以是叶子或组合组：

```json
{
  "documentVersion": 3,
  "activeSlideId": "slide-1",
  "slides": [{
    "id": "slide-1",
    "name": "Page 1",
    "width": 1080,
    "height": 1440,
    "background": { "type": "solid", "color": "#ffffff" },
    "nodes": [{
      "id": "group-1",
      "name": "标题组",
      "type": "group",
      "x": 240,
      "y": 180,
      "rotation": 0,
      "scale": 1,
      "locked": false,
      "hidden": false,
      "children": []
    }]
  }]
}
```

叶子类型为 `text`、`image`、`shape` 和 `line`。叶子使用父级坐标系中的 `x/y`、`width/height`、顺时针 `rotation` 和统一 `scale`。组只有原点、旋转、统一缩放和子节点；组的视觉边界由所有后代节点的世界坐标边界计算，不能把旋转后的轴对齐外框当作本地几何。

编辑器限制如下：

- 页面尺寸为 128 到 4096 px，最多 500 个页面。
- 单页最多 5000 个场景节点，递归深度最多 32 层。
- 后代世界有效缩放必须在 `1e-4` 到 `1e4` 之间。
- 组只支持统一缩放；叶子仍可按宽度和高度分别编辑。
- 所有矩阵运算保持浮点精度，属性面板只在显示时保留两位小数。

## 迁移和持久化

读取文档时，v1/v2 平面 `slides[].elements` 会先复制到新的节点树，再经过严格 v3 归一化。迁移会补齐稳定 ID、过滤坏节点、校验尺寸/数量/深度/缩放，并把 `documentVersion` 写成 3。输入对象不会被原地修改；无法生成合法 v3 结果的文档会被拒绝，不会以未校验状态进入工作区。

保存时本地和远程存储都只接受 v3 文档。远程保存会在提交前递归处理隐藏节点中的图片引用，历史 Data URL 会上传并替换为托管 URL；图片引用扫描不会因为节点隐藏而跳过。远程保存响应经过同一归一化边界后才回到工作区。

## 图层面板

右侧面板有“属性”和“图层”两个标签。图层标签使用可访问的递归树：

- 单击选择节点；`Shift`、`Ctrl`/`Cmd` 可在同一父级内多选。
- 组可展开、折叠和重命名；树行保留层级、当前选择和继承状态。
- “组合”要求至少两个同级节点；“解组”按一层提升子节点。锁定、隐藏、跨父级或超出深度/数量限制的操作会被拒绝且不产生历史记录。
- 树行的锁定和隐藏按钮只改变节点自身状态。父级锁定或隐藏会向后代继承；后代仍可在面板中查看自身状态和继承原因。
- 树键盘支持方向键、`Home`、`End`、`Enter`、空格、`F2` 和 `Alt + ↑/↓`：`Enter` 单选，空格切换多选，`F2` 重命名，左右键展开/折叠，`Alt + ↑/↓` 调整同级顺序。画布双击组内节点或对已选组按 `Enter` 可进入编辑作用域，`Escape` 逐层退出；文本编辑和弹层会先消费自己的 `Escape`。

常用快捷键：

| 操作 | 快捷键 |
|---|---|
| 组合 | `Ctrl/Cmd + G` |
| 解组 | `Ctrl/Cmd + Shift + G` |
| 复制/粘贴 | `Ctrl/Cmd + C` / `Ctrl/Cmd + V` |
| 撤销/重做 | `Ctrl/Cmd + Z` / `Ctrl/Cmd + Shift + Z` 或 `Ctrl/Cmd + Y` |
| 删除 | `Delete` 或 `Backspace` |
| 方向微调 | 方向键；`Shift` 加大步长 |

## 变换和剪贴板

画布指针、框选、吸附、方向微调、对齐和分布都在页面世界坐标中计算，再通过父级世界矩阵转换回本地坐标。组作为一个逻辑单位移动、旋转、缩放和对齐，组内子节点的相对变换保持不变。有效锁定节点不会出现变换控件，也不能被键盘或指针修改。

复制内容在内存中保留来源父级的世界矩阵。粘贴到另一个作用域时，目标节点的世界位置等于来源世界位置加页面偏移 `(16, 16)`，随后转换为目标父级本地坐标；嵌套 ID 会递归重建且不会与文档现有 ID 冲突。非法有效缩放会拒绝整次粘贴，不会破坏剪贴板。

## 图片、导出和历史

本地模式把图片暂存为会话引用，保存副本时物化为 Data URL。远程模式把图片上传到 `/api/images`，以托管 URL 写入草稿，并通过 retain/租约机制续期。服务端只回收“租约已过期且没有任何草稿引用”的图片，因此隐藏组中的图片也会保留；集成测试使用 `IMAGE_LEASE_MS=500` 加速验证，生产默认租约不变。

PNG 导出只渲染画布艺术层，不包含选择框、旋转手柄、图层树或属性面板。隐藏节点及其后代不产生像素，锁定节点仍参与渲染。导出尺寸始终等于页面逻辑尺寸，与浅色/深色主题和预览缩放无关。

保存响应可能在实时指针手势期间返回。工作区使用统一历史入口和保存快照权威门：`pointerup` 只提交一次手势历史，`pointercancel` 恢复对应保存快照和已保存状态；旧草稿或旧用户的迟到响应不会改变当前作用域、选择或草稿身份。

## 验证入口

```powershell
npm run test:unit
npm run test:server
npm run test:e2e
npm run test:acceptance
npm run test:integration
npm run build
```

`test:integration` 不连接用户正在使用的预览服务：集成后端使用 `5310`，前端使用 `5273`，端口定义集中在 `e2e-integration/ports.ts`。集成配置会为每次运行创建临时数据目录，并只在该配置中把 `IMAGE_LEASE_MS` 设置为 `500`。验收测试覆盖 1440、1366 和 1024 宽度布局、原生控件检查、嵌套图层保存/重载以及 5000 ms 导出上限。
