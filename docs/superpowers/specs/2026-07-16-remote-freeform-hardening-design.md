# 远程自由编辑接入加固设计

## 1. 目标

在不改变本地模式默认可用性的前提下，关闭真实后端模式中会影响自由编辑继续演进的数据完整性风险：

- 草稿更新必须严格按当前用户隔离，不能凭全局草稿 ID 覆盖其他用户数据。
- Markdown 与自由编辑插图统一通过 `ImageStore`，远程模式只在草稿中保存稳定 URL，不保存 Base64。
- 本地模式仍可离线工作，保存自由编辑草稿时必须把临时图片引用物化为可持久化数据。
- 网络失败、401、无效响应和浏览器存储不可用时，认证与草稿操作行为必须可解释、可恢复。
- 本地与远程草稿读取使用同一套运行时归一化契约。
- 浏览器自动化测试不得依赖 Google Fonts 可访问。
- 新增能力、脚本、错误行为、版本号与文档同步。

## 2. 非目标

- 本批不自动上传已有 localStorage 草稿。自动迁移可能在用户不知情时上传本机内容；后续应提供显式“导入本机草稿”流程。
- 不实现邮件、协作、分享链接或新的自由编辑 UI 功能。
- 不替换 DOM 导出架构。
- 不承诺本批完成完整媒体资产管理后台；只实现安全、可回收的当前图片生命周期。

## 3. 草稿所有权

后端保留全局唯一草稿 ID，但 upsert 必须同时满足 `id + user_id`：

1. 无 ID 时由服务端生成 ID 并插入。
2. 有 ID 且属于当前用户时更新。
3. 有 ID 但不存在或属于其他用户时都返回 404，避免泄露资源是否存在；带 ID 的请求不创建新草稿。
4. SQL 更新语句自身也必须带用户条件，防止检查与写入之间的竞态。

可选 `id` 只接受非空字符串；传入空字符串、非字符串或仅空白字符时返回 400，不能被当成“无 ID”静默新建。

删除接口保持幂等：目标不存在或属于其他用户时仍返回成功，但 SQL 必须带 `id + user_id`，不得泄露资源归属。

服务端 smoke test 增加“用户 B 使用用户 A 草稿 ID 保存”场景，验证返回 404 且 A 的草稿内容不变。

## 4. 图片数据流

### 4.1 插入

Markdown、自由编辑图片元素、形状图片填充都执行：

1. 浏览器读取文件并降采样。
2. 调用 `store.images.put(dataUrl)`。
3. LocalStore 返回 `img:<id>`；RemoteStore 上传 `/api/images` 并返回稳定 URL。
4. 画布渲染统一调用 `store.images.resolve(src)`。

### 4.2 本地草稿持久化

自由编辑活动文档可持有 `img:` 临时引用。LocalStore 保存草稿前克隆文档，把图片元素和形状图片填充中的引用解析为 Data URL；不修改当前画布内存文档。这样草稿跨会话可恢复，同时避免编辑期间在 reducer/history 中反复复制大 Base64。

只处理 `img:` 引用；已有 Data URL、相对 URL 和绝对 URL 原样保留。任一 `img:` 引用无法解析时，本次保存整体失败且不覆盖已有草稿，不能静默保存一份重载后丢图的文档。

RemoteStore 保存前还要处理升级前已经存在于远程草稿中的 Data URL：克隆自由编辑文档，把图片元素和形状图片填充中的 `data:image/...` 逐个上传并替换为服务器 URL，同一 Data URL 在单次保存中只上传一次。任一上传或草稿提交失败时不缓存替换关系，原活动文档保持不变；下次保存重新上传，不能复用可能已被回收的 URL。完整保存成功后，工作区仅在用户尚未继续编辑该保存快照时，才用服务端返回的 URL 文档替换当前文档；若保存期间已有新编辑则不覆盖新状态。`img:` 属于 LocalStore 私有引用，RemoteStore 无法解析时明确拒绝保存。这样历史远程草稿再次保存后也只包含服务器 URL，并且成功后的活动文档会参与正常续租。

### 4.3 清理

服务端增加“已保存引用 + 活跃租约”共同保护的用户级孤儿图片回收，不能只依赖图片创建时间：

- `images` 增加 `lease_expires_at`。数据库启动迁移必须检测旧表是否缺列并执行 `ALTER TABLE`；旧图片统一回填“迁移时刻 + 一个完整租约”，不能因升级立即被回收。新上传成功后立即获得租约。
- 远程 `ImageStore` 增加 `retain(urls)`。工作区对活动文档中的图片在首次出现、浏览器 `online`、页面重新可见和固定间隔内续租；保存、删除草稿以及上传下一张图片之前也必须先续租活动图片，避免离线超过租约后刚恢复就由自己的操作先触发 GC。
- 租约时长由 `IMAGE_LEASE_MS` 配置，默认 24 小时；客户端续租间隔固定为 5 分钟。测试可把租约缩短或直接构造已过期记录。
- 从该用户所有草稿 JSON 中递归收集 `/uploads/...` 引用。相对 `/uploads/...` 与同一路径的绝对 URL 都视为同一引用；比较时使用规范化后的 URL pathname，不按字符串前缀猜测。
- 只有“未被任何已保存草稿引用且租约已过期”的图片才可删除。已保存引用没有过期时间；活动但未保存、或在其他标签页/设备中打开的内容由续租保护。
- 草稿保存、草稿删除、图片续租与上传按用户串行执行，避免“扫描完成后并发保存新增引用”的检查—删除竞态；其中上传锁必须覆盖“回收 → 配额判断 → 写文件 → 插入图片记录”整个临界区，防止并发上传同时通过配额检查。不同用户之间不互相阻塞。
- 删除草稿后和配额检查前触发回收。任一草稿 JSON 无法解析时，本轮回收保守退出，不删除该用户的任何图片。
- 文件删除失败不得删除数据库记录；文件已不存在可视为删除成功。数据库记录删除失败时保留记录，后续回收可重试。
- 图片文件写入成功但数据库记录插入失败时，立即补偿删除刚写入的文件；补偿失败记录服务端错误日志，不能返回伪成功。

租约解决“已上传但尚未保存”以及“草稿在另一端被删除、当前画布仍打开”的生命周期空窗。配额错误文案说明清理只会回收租约已过期的未引用图片，因此释放空间可能有延迟。

`retain(urls)` 只管理当前服务 `UPLOADS_PUBLIC_PATH` 下的 URL，外部 URL 与 Data URL 忽略。请求中的托管路径必须全部存在且属于当前用户；否则整次续租返回不区分原因的 409，不能以部分成功伪装完整保护，也不能泄露其他用户资源。后台定时续租失败时捕获错误、显示非阻塞警告并在较短间隔重试；保存、删除或上传前的安全续租失败时阻止该操作并显示错误，绝不继续触发可能回收图片的请求。

## 5. 远程错误与认证

远程请求使用带 `status: number | null` 的 `ApiError`；网络层没有 HTTP 响应时 `status` 为 `null`：

- 仅 401 可触发 token 失效，而且只能清除该请求实际携带、并且仍是当前值的 token；旧请求迟到的 401 不得清除之后登录得到的新 token。
- 注册和登录属于公共认证请求，不携带已有 bearer token，其 401 只表示本次凭据失败，不能触发现有会话失效。只有实际携带 bearer token 的受保护请求才执行上述 401 失效逻辑。
- 网络失败、5xx、无效 JSON 不清除 token，并向调用方抛出稳定、可显示的错误。401 即使响应体不是有效 JSON 也必须清除 token。
- token 同时保存在模块内存；localStorage 被禁用时当前页面仍可继续登录态。
- `auth.current()` 仅在没有 token 或收到 401 时返回 `null`；网络、5xx 和无效响应继续抛错。
- `AuthStore` 提供认证失效订阅。当前 token 因 401 被清除时，RemoteStore 通知订阅者；`AppShell` 立即清空对应用户状态并显示重新登录入口。LocalStore 提供行为一致的空订阅实现。
- `AppShell` 显式维护认证检查状态和带重试入口的非阻塞错误提示，避免把网络故障伪装成退出登录。
- 两个工作区的草稿列表、保存、删除和图片上传都捕获错误并显示；失败时保留上一次成功状态，不产生未处理 Promise rejection。

## 6. 草稿归一化

LocalStore 与 RemoteStore 的 `list`、`save` 返回值都必须经过 `normalizeDraftForRead`：

- 旧 Markdown 草稿与自由编辑 v1/v2 行为一致。
- `list` 的顶层响应不是数组时抛出明确错误；数组中的无效单项按 LocalStore 现有契约丢弃，其余有效草稿正常返回。
- `save` 的单项响应无法归一化时抛出明确错误，不能直接进入 reducer。
- 服务端不理解工作区 schema，也不承担前端文档迁移；图片回收仅对任意 JSON 字符串值做通用 URL pathname 扫描，不依赖 Markdown 或自由编辑字段结构。

## 7. 测试

按 TDD 添加以下覆盖：

- 后端跨用户草稿覆盖拒绝及原数据不变。
- 自由编辑图片元素与形状填充调用 ImageStore，并保存 URL 而非 Base64。
- LocalStore 保存自由编辑草稿时物化图片引用，原文档保持不变。
- LocalStore 遇到缺失 `img:` 引用时保存原子失败，不覆盖已有草稿。
- RemoteStore 保存历史 Data URL 自由编辑草稿时上传并替换图片，原文档保持不变，失败时不提交草稿。
- 历史 Data URL 草稿提交失败后再次保存必须重新上传，不能复用上次失败遗留的 URL；成功后仅在活动快照未发生后续编辑时采用服务器 URL 文档。
- RemoteStore 仅在 401 且请求 token 仍为当前值时清 token并通知认证失效；网络/5xx 保留；存储被阻止时使用内存 token；无效 JSON 返回稳定错误；迟到的旧 token 401 不影响新登录。
- RemoteStore 归一化旧草稿并拒绝无效响应。
- 图片回收覆盖：相对/绝对 URL 引用、已保存引用保留、有效租约保留、过期孤儿删除、损坏 JSON 保守退出、文件不存在、文件删除失败、数据库删除失败、写文件后插库失败补偿，以及同用户保存/回收串行化。
- 图片生命周期还覆盖：旧数据库加列与安全回填、完整上传临界区串行、离线超过租约后恢复并在上传前先续租、后台续租失败重试、保存/删除/上传前续租失败阻断，以及托管 URL 部分缺失时整次续租失败。
- 真实后端联调覆盖自由编辑图片上传、保存、重载恢复和草稿 JSON 无 Base64。
- 真实后端联调还要实际导出包含普通图片和形状图片填充的页面，不能只检查 DOM URL。
- 认证测试覆盖登录失败的公共 401 不清现有 token，以及迟到的受保护请求 401 不清新 token。
- Playwright 在测试层拦截外部字体，离线环境仍可完成导航；联调拓扑必须能从真实后端读取测试上传的 `/uploads/...` 文件。

验证命令：

```text
npm run test:unit
node server/smoke-test.mjs
npm run test:integration
npm run test:e2e
npm run build
docker compose config
```

## 8. 版本与文档

- 根应用由 `0.8.0` 升至 `0.9.0`。
- 后端由 `0.1.0` 升至 `0.2.0`。
- 同步两个 package lock。
- `docs/backend-plan.md` 改为实际的 `src/storage/*` 架构，记录自由编辑图片路径、状态码、回收语义和测试命令。
- 根 `.env.example`、`server/.env.example`、`docker-compose.yml` 与部署文档同步 `IMAGE_LEASE_MS`，保证 Docker 环境确实把配置传入后端。

## 9. 完成标准

- 任何用户不能覆盖、读取或删除其他用户草稿。
- 远程自由编辑草稿不含 `data:image/...`。
- 含普通图片和形状图片填充的作品重载后可恢复并导出。
- 网络故障不会静默清除有效 token，保存失败有可见反馈。
- 本地模式原有草稿和图片行为不回退。
- 自动化测试在无外网字体环境下可重复运行。
