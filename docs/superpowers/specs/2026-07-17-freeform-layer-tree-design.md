# 自由编辑图层树与交互加固设计

日期：2026-07-17
分支：`feature/freeform-layer-tree`
状态：用户授权自检后继续，三轮 spec review 问题已修订，设计定稿

## 1. 背景

自由编辑模式已经支持文本、图片、形状、线条、多选、对齐、吸附、基础层级、草稿与导出，但文档仍是扁平元素数组。自动化探索验收还发现了四类交互缺口：字体选项点击后菜单未收起、`Esc` 关闭字体菜单会同时清空画布选择、认证弹窗缺少完整对话框语义，以及随画布缩放后变得过小的移动/缩放命中区。

下一阶段要补齐两组能力：

1. 先修复上述交互与无障碍问题，并把探索验收固化进 Playwright。
2. 增加右侧图层树、锁定、隐藏、组合与解组；组合支持嵌套，并具备 PPT 核心级整体移动、等比缩放、旋转、复制和层级调整。

当前自由编辑文档为 `documentVersion: 2`，页面通过 `slide.elements` 的数组顺序表达从底到顶的层级。继续叠加扁平 `groupId` 会让嵌套、继承状态和组合变换逐渐失去确定性，因此本设计将自由编辑文档升级为递归场景树。

## 2. 已确认的产品决策

- 图层面板采用方案 A：右侧栏在“属性 / 图层”两个标签页间切换；左侧页面栏保持不变。
- 组合支持嵌套，不采用只能容纳一级成员的临时 `groupId`。
- 第一次点击组内内容时选中当前编辑范围内的最外层组；双击或按 `Enter` 进入组；`Esc` 逐层退出，最后才清空选择。
- 图层面板可以直接选择任意层级节点，并同步进入该节点的父级编辑范围。
- 组支持整体移动、等比缩放和旋转；不支持倾斜、透视和 3D 变换。
- 锁定和隐藏沿祖先向下生效，但不覆盖子节点自身保存的状态。
- 所有文档操作必须可撤销、可保存、可远程恢复，并保持导出结果确定。

## 3. 目标与非目标

### 3.1 目标

- 修复字体菜单、认证弹窗与变换手柄的已知交互问题。
- 提供 224–248px 宽的右侧图层树，适配 1024、1366、1440 桌面视口和明暗主题。
- 支持节点命名、树形展开、选择、多选、同级排序、锁定和隐藏。
- 支持任意重复嵌套的组合与解组；读取不可信草稿时设置安全深度上限。
- 组合后支持整体移动、方向键微调、等比缩放、旋转、复制、删除、对齐、吸附和层级调整。
- 保证旧 v1/v2 自由编辑草稿无损迁移到 v3。
- 保证隐藏节点不显示、不导出；锁定节点仍显示并导出。
- 保证隐藏图片仍被资产收集与续租，避免远程图片被回收。
- 将真实浏览器验收固化为可重复运行的 Playwright 测试。

### 3.2 本批不做

- 非等比组缩放、倾斜、透视与 3D。
- 跨父组拖拽重挂节点；图层拖拽只允许同级排序。
- 组内富文本局部样式升级。
- 图层搜索、过滤、批量重命名和图层颜色标签。
- 实时协作、多人选择状态和云端版本历史。
- 更换 DOM 渲染为 Canvas/Fabric/Konva。

## 4. 方案比较

### 4.1 数据模型

#### 方案一：扁平元素 + 单层 `groupId`

改动最少，但无法自然表达组中组。后续升级需要再次迁移草稿、剪贴板、选择和层级语义，因此不采用。

#### 方案二：扁平节点表 + `parentId`

可以表达嵌套，查询单节点方便，但每次渲染、复制、删除、排序和继承状态都要重建父子关系；损坏的 `parentId` 还会产生孤儿和环。适合数据库实体，不适合当前内存文档。

#### 方案三：递归场景树 + 局部变换（采用）

父子关系与层级顺序直接由树结构表达，复制、删除、锁定隐藏继承和导出遍历都具有确定语义。代价是需要矩阵与递归帮助函数，但这是完整嵌套组合必须承担的复杂度。

### 4.2 图层面板位置

#### 方案 A：右侧“属性 / 图层”标签（采用）

嵌套树拥有 224–248px 可用宽度，1024px 下仍能阅读节点名称；左侧页面栏职责不变，画布列宽不增加第四栏。

#### 方案 B：左侧“页面 / 图层”标签

可以同时查看图层和属性，但现有 136–152px 左栏不足以显示多级嵌套，扩宽后会明显压缩紧凑视口中的画布，因此不采用。

## 5. 文档 v3 场景树

### 5.1 核心类型

自由编辑草稿信封继续使用 `schemaVersion: 2`；自由编辑文档内部升级为 `documentVersion: 3`。

```ts
interface FreeformDocument {
  documentVersion: 3
  slides: FreeformSlide[]
  activeSlideId: string
}

interface FreeformSlide {
  id: string
  name: string
  width: number
  height: number
  background: SlideBackground
  nodes: FreeformSceneNode[]
}

interface SceneNodeState {
  id: string
  name: string
  locked: boolean
  hidden: boolean
}

type FreeformSceneNode = FreeformElement | FreeformGroupNode

interface FreeformGroupNode extends SceneNodeState {
  type: 'group'
  /** 组原点在父坐标系中的位置；组内坐标以该原点为中心。 */
  x: number
  y: number
  rotation: number
  /** 只允许正数等比缩放。 */
  scale: number
  children: FreeformSceneNode[]
}
```

现有文本、图片、形状和线条元素继续保存 `x/y/width/height/rotation`，并增加 `name/locked/hidden/scale`。根节点元素的 `x/y` 仍是页面内左上角坐标；组内叶子元素的 `x/y` 是相对父组原点的左上角坐标。叶子旋转和等比缩放都围绕自身中心。

叶子 `scale` 不是可见的独立属性控件，而是无损保留组合变换的内部字段。它会同时缩放尺寸、字号、描边、箭头、内边距和圆角等所有视觉长度，避免组缩放后解组改变像素外观。几何输入按第 7.4 节显示父局部坐标与沿叶子本地轴的缩放后尺寸；用户修改宽高时只通过叶子 scale 换算回基础 `width/height`。v1/v2 元素迁移时 `scale=1`。

### 5.2 变换与矩阵契约

- 组原点位于其后代局部包围盒中心。
- 组移动只修改 `x/y`。
- 组旋转只修改 `rotation`。
- 组缩放只修改正数 `scale`，通过角部手柄等比缩放。局部持久化 `scale` 只要求正且有限；严格校验要求每个节点从页面根累计得到的世界有效缩放位于共享范围 `MIN_EFFECTIVE_SCALE=1e-4` 到 `MAX_EFFECTIVE_SCALE=1e4`。该约束对组合/解组的矩阵合成闭包：解组改变局部 scale 的分配，但不改变任何节点的世界有效缩放。
- 组合时计算所选同级节点的父坐标包围盒中心，把节点转换成组内局部坐标，世界几何保持不变。
- 解组时把父组的平移、等比缩放和旋转合成到直接子节点：子组的 `scale` 与父组相乘；叶子的 `scale` 与父组相乘。所有视觉长度继续由节点 scale 表达，因此不产生倾斜或样式漂移。
- 画布缩放或属性输入改变组 scale 前，计算该组及全部后代的新世界有效缩放，并把操作量夹在共享有效范围内；组合、解组和自动解散单子组不改变世界有效缩放，因此不会因重新分配局部 scale 被拒绝。
- 子节点在组内被编辑后，从内向外重新居中受影响的祖先组；重心移动由父级坐标补偿，世界几何不得跳动。
- 组合和解组各自产生一条历史记录。

矩阵使用列向量、页面坐标系（x 向右、y 向下）和顺时针正角度，组合顺序固定为：

```ts
GroupLocal = T(x, y) * R_clockwise(rotation) * S(scale)
LeafLocal =
  T(x + width / 2, y + height / 2) *
  R_clockwise(rotation) *
  S(scale) *
  T(-width / 2, -height / 2)
World(node) = World(parent) * Local(node)
```

叶子内容在自己的 `[0,width] × [0,height]` 局部矩形中渲染。包围盒由四角乘完整世界矩阵后取 min/max 得到；中间计算和持久化不做整数舍入，属性 UI 最多显示两位小数。几何比较统一使用 `SCENE_EPSILON = 1e-6`。

组重新居中公式固定为：先计算直接后代几何的局部联合包围盒中心 `c`，把每个直接子节点的父坐标位置减去 `c`，再令：

```ts
groupPosition' = groupPosition + linear(R(rotation) * S(scale)) * c
```

这样所有后代的世界矩阵保持不变。实现必须用矩阵测试验证重居中前后每个叶子四角误差不超过 epsilon。

指针先按现有预览缩放从屏幕坐标转换为页面坐标，再乘 `inverse(World(activeParent))` 得到当前编辑范围的父局部坐标；位移向量只乘逆矩阵的线性部分，不包含平移。框选、拖动、缩放、微调、吸附和对齐都使用这一条坐标转换链路，不再直接把页面 delta 写入组内节点。

### 5.3 安全边界

- 导出共享常量 `MAX_SCENE_DEPTH=32`、`MAX_SCENE_NODES_PER_SLIDE=5000`、`MAX_FREEFORM_SLIDES=500`。根节点深度为 1，组本身和叶子都计入节点总数。
- reducer 在组合、粘贴和新增页面前检查上限；超限时返回原文档引用并显示非阻断提示。应用自身不得创建保存后无法读取的文档。
- v3 必须包含 1–500 张页面；每页宽高都必须通过现有 `validatePageSize`（整数且 128–4096），slide ID 非空且文档内唯一，`activeSlideId` 必须准确指向其中一页。空页面数组、非法尺寸和失效活动页都原子拒绝，不回退。
- v3 归一化对深度、节点数、页面数、页面 invariant、页内重复/空 node ID、非有限几何、非正叶子尺寸、非正/非有限局部 scale，以及超出共享范围的世界有效缩放采用原子拒绝：整个自由编辑文档返回 `null`，不得静默截断并在下一次保存覆盖原数据。
- v1/v2 迁移继续兼容既有的“跳过单个损坏元素”行为，但输出必须立即满足全部 v3 invariant：
  - 非正宽高或其他必需字段损坏的旧元素被跳过；有限的负 `x/y` 继续保留。
  - 空或同页重复 node ID 按页面与原数组顺序确定性改写为 `legacy-node-<slideIndex>-<nodeIndex>`，冲突时追加递增后缀；不得丢掉后出现的有效元素。
  - 空或重复 slide ID 按页面顺序确定性改写为 `legacy-slide-<index>` 并追加冲突后缀；slide ID 在文档内唯一。
  - 页面尺寸不满足现有 128–4096 整数契约时跳过该页；全部页面被跳过则迁移失败。
  - `activeSlideId` 指向重复旧 ID 时使用第一个匹配页；无匹配时回退第一张有效页。
  - 全局上限超出时拒绝整个文档，不截断。
- 损坏组不提升其子节点，因为组变换缺失时无法安全保留世界几何。
- v3 空组属于结构错误并原子拒绝整份文档；单子节点组允许读取以避免不必要的拒绝。正常 reducer 在删除操作后用与显式解组相同的矩阵合成规则自动解散单子节点组，删除空组。
- v1/v2 的 `elements` 按原数组顺序迁移为根级 `nodes`，默认名称按类型派生，`locked=false`、`hidden=false`。
- 节点 ID 只要求在单页内唯一。当前旧版本复制页面可能在不同页面保留相同元素 ID，迁移不得进行跨页去重。v3 新建和复制页面仍为所有节点生成新 ID，作为卫生保证而非读取前提。迁移测试必须执行“读取 v1/v2 → 保存 v3 → 再次读取”，第二次读取不得丢页、丢节点或改变新 ID。
- 所有归一化路径统一输出 v3；LocalStore 和 RemoteStore 行为一致。

## 6. 场景树纯逻辑边界

新增聚焦的纯逻辑模块，而不是继续扩大 `FreeformWorkspace.tsx`：

- `sceneTree.ts`：遍历、查找路径、按路径更新、深拷贝、同级排序、组合、解组、删除和退化组清理。
- `sceneTransform.ts`：二维矩阵、世界变换、包围盒、组合/解组坐标换算、组等比缩放和祖先重新居中。
- `sceneSelection.ts`：编辑范围、选区归一化、祖先/后代去重、有效锁定与隐藏、逻辑选择单元。
- `FreeformSceneNodeView.tsx`：递归渲染组和叶子元素。
- `FreeformLayersPanel.tsx`：图层树及其键盘、锁定、隐藏、命名和排序交互。
- `FreeformSelectionOverlay.tsx`：画布级选择框和命中区，不改变作品层级。

关键函数必须满足稳定契约：

- 未知路径、空选择、非同级组合、边界排序，以及被有效锁定节点的内容、几何、样式或结构变更返回原文档引用；下方权限矩阵明确允许的图层元数据动作除外。批量动作只要包含一个无权限目标就整批原子拒绝。
- 查询函数不修改输入。
- 更新一个路径只复制该路径上的祖先，未受影响分支保持引用。
- 深拷贝为所有组和叶子生成新 ID，组内图片引用保持不变。
- reducer 是锁定规则的最终防线，不能只依赖 UI 禁用。

组合允许选择不连续的同级节点。新组插入到“原来位于最高层的选中节点”所在视觉层级：移除所有选中节点后，在仍位于该最高节点下方的未选中节点之后插入组；组内 children 保持原数组相对顺序。由于一个组必须成为连续绘制子树，夹在选中节点之间的未选中兄弟与部分组选中项之间的遮挡关系可能改变，这是明确接受的组合语义。解组时在组当前位置按 children 顺序回插，不尝试恢复组合前已经改变的交错层级。

锁定权限由 reducer 按动作分类执行：

| 动作 | 自身锁定 | 仅祖先导致有效锁定 | 锁定父容器 |
|---|---|---|---|
| `node/set-locked` | 允许，用于解锁 | 允许修改自身字段，但有效状态仍为锁定 | 允许 |
| `node/set-hidden` | 允许 | 允许 | 允许 |
| `node/rename` | 允许 | 允许 | 允许 |
| 内容/几何/样式修改 | 拒绝 | 拒绝 | 拒绝 |
| 删除、排序、组合、解组 | 拒绝 | 拒绝 | 拒绝 |
| 向容器粘贴或新增子节点 | 不适用 | 不适用 | 拒绝 |

锁定用于保护作品内容，图层管理元数据仍可操作；隐藏按钮因此也可以作用于锁定节点。组合要求至少两个同父级、未有效隐藏、未有效锁定的节点，且父容器本身可编辑。

## 7. 选择与编辑范围

### 7.1 编辑范围

工作区增加 `activeGroupPath`，表示当前允许直接选择的父容器：

- 根范围中点击组内任意可见内容，选择根级组。
- 双击组或按 `Enter` 进入该组，选区清空，面包屑显示当前位置。
- 组内点击子组先选子组；继续双击或 `Enter` 可逐层进入。
- `Esc` 先关闭当前浮层或内联编辑；否则退出一层组；已经在根范围时才清空选区。
- 图层面板直接选择深层节点时，将编辑范围切到该节点父级并选择该节点。
- `activeGroupPath` 和选区属于 UI 状态，不进入历史。每次当前文档快照变化后都统一校验，包括 reducer、undo/redo、草稿打开、迟到远程保存响应、切页和删除祖先组；无效路径回退到最近仍存在的祖先，完全不存在时回到根，并过滤选区。

### 7.2 多选

- 多选只包含同一父容器的直接子节点。
- 图层树尝试跨父级 Shift/Ctrl 多选时，保留原选择并通过非阻断提示说明“只能同时选择同一组内的图层”。
- 选择组与其后代不能同时存在；选择归一化始终保留最高层节点。
- 框选只命中当前编辑范围内未有效隐藏、未有效锁定的直接子节点。
- 隐藏当前选中节点后保留图层行选择，但移除画布 overlay；用户仍可通过同一行恢复显示。

### 7.3 变换、对齐和吸附

- 单个组使用其局部包围盒和世界矩阵显示旋转选择框。
- 多选显示一个世界坐标联合包围框；移动保持成员相对关系。
- 组与多选的缩放都使用角部手柄做等比缩放。
- 对齐与分布按所选逻辑节点的世界包围盒计算，对组只施加整体位移，不改变内部布局。
- 隐藏节点不参与吸附；锁定但可见节点可作为吸附参考。
- 方向键移动 1px，Shift + 方向键移动 10px；锁定选择不响应。

### 7.4 属性面板坐标语义

属性面板不把旋转后的世界 AABB 冒充节点宽高。所有可编辑值都相对于直接父容器，面包屑明确显示“页面”或“组内”：

- 叶子 `X/Y`：父局部坐标中的未旋转基础矩形左上角。
- 叶子 `宽/高`：沿叶子自身局部轴的缩放后尺寸，即 `width * leaf.scale`、`height * leaf.scale`，不乘祖先 scale，也不受 rotation 改变。编辑宽或高时分别反算基础 `width/height`，叶子 scale 不变。
- 叶子 `旋转`：相对直接父容器的本地 rotation。叶子 scale 不直接暴露；字号和描边等视觉输入显示并编辑 `baseValue * leaf.scale`，写入时除以 leaf scale。
- 组 `中心 X/Y`：组原点在直接父坐标系中的位置。
- 组 `宽/高`：组局部后代联合包围盒乘本组 scale 后，沿组自身局部轴的尺寸；两者保持比例联动。编辑任一项只调整组 scale，另一项自动更新。
- 组 `旋转` 与 `缩放 %`：相对直接父容器的本地值。缩放输入通过全部后代的世界有效缩放范围校验。
- 旋转后的世界 AABB 只用于命中、框选、吸附和 overlay，不在属性输入中展示。

## 8. 锁定与隐藏

有效状态由自身和祖先共同决定：

```ts
effectiveHidden = node.hidden || ancestors.some((node) => node.hidden)
effectiveLocked = node.locked || ancestors.some((node) => node.locked)
```

- 隐藏节点及其后代不进入作品 DOM、不导出、不接受命中、不参与吸附。
- 隐藏图片仍由资产收集器递归收集并续租。
- 锁定节点继续渲染、导出并可作为吸附目标，但不能通过画布选择、拖动、缩放、旋转、文字编辑、删除、粘贴覆盖或层级调整。
- 锁定节点可以从图层树选中；属性面板显示“已锁定”只读横幅，并保留解锁入口。
- 修改父组状态不重写子节点字段；解除父组状态后，子节点恢复自身状态。
- 锁定和隐藏切换各自产生一条可撤销历史。

## 9. 右侧图层面板

### 9.1 布局

- 右侧栏顶部使用语义化 `tablist`，包含“属性”和“图层”。
- tabs 使用 roving tabindex，完整同步 `role="tab"`、`aria-selected`、`aria-controls` 与对应 `role="tabpanel"`；方向键、Home/End 切换，焦点留在活动 tab。
- 初次进入默认显示属性；用户本会话内的选择保存在 UI 状态，不写草稿、不进入历史。
- 图层树按作品层级逆序显示，即视觉最上层位于列表最上方。
- 行高 40px，类型、展开、名称、显示和锁定使用统一 SVG 图标，不使用 emoji。
- 选中态使用 `accent-weak` 背景和 3px 陶土色左边线；悬停使用中性表面。

### 9.2 行为

- 组节点可展开/折叠；折叠状态仅属于 UI。
- 单击行选择节点；Shift/Ctrl/Meta 在同一父级多选。
- 双击名称或按 `F2` 进入重命名；空名称提交时回退到类型默认名。
- 眼睛和锁按钮使用 `aria-pressed`、明确名称并阻止画布快捷键处理。
- 鼠标拖拽只允许同级重排，不允许跨父容器重挂。
- 图层数组从底到顶，面板从顶到底。`Alt+↑` 表示视觉上移/更靠前/数组索引增加，`Alt+↓` 表示视觉下移/更靠后/数组索引减少；多选按稳定块移动并保持内部相对顺序。拖拽落点先按视觉顺序计算，再转换为数组位置。
- 提供 `Alt+↑/↓` 键盘同级排序，并通过 `aria-live` 宣布节点名称和新位置；鼠标不是唯一途径。
- 树容器使用 `role="tree"` 与 `aria-multiselectable="true"`；节点使用 `role="treeitem"`、`aria-level`、`aria-expanded`、`aria-selected`，子级容器使用 `role="group"` 和 roving tabindex。
- 上下键移动可见树项焦点，左右键展开/折叠或进入/返回父节点，Home/End 跳至首尾；Enter 选择当前项，Space 按同父级多选规则切换选择。
- 删除、隐藏或折叠当前焦点项后，焦点依次落到同级下一项、同级上一项或父项；重排后焦点跟随被移动节点；切换 tab 后重新进入图层 tab 时恢复最近仍存在的树项。
- 上下文操作条显示“组合 / 解组”；同时支持 `Ctrl/Cmd+G` 与 `Ctrl/Cmd+Shift+G`。

## 10. 渲染与选择 UI

- 作品节点严格按树中兄弟顺序渲染；选中节点不得通过 `z-index` 临时抬到顶层。
- 组使用嵌套 DOM 与 CSS 变换渲染，导出仍复用现有 DOM 截图链路。
- 选择框、拖动区、缩放和旋转手柄移动到独立 `freeform-ui-only` overlay，在全部作品节点之后渲染。
- overlay 根据节点世界矩阵定位；单选组显示旋转框，多选显示联合包围框。
- 控制点视觉可保持克制，但屏幕命中区至少 28×28px，不随画布缩放变小；通过反向缩放 CSS 变量实现。
- 锁定文本必须让 `PlainTextEditable` 进入只读模式，不能只依赖父层 `pointer-events`。
- 导出过滤 overlay，明暗主题和预览缩放不得改变导出像素。

## 11. 剪贴板、页面复制与资产

- 内部剪贴板保存完整 `FreeformSceneNode[]`、复制时按原数组顺序排列的顶层选择，以及复制瞬间的 `sourceParentWorld` 相似变换矩阵。
- 复制组递归复制全部后代。粘贴为所有节点生成新 ID，并把每个顶层节点从源父坐标系转换到目标父坐标系：

```ts
PastedLocal = inverse(TargetParentWorld) * T_world(16, 16) * SourceParentWorld * SourceNodeLocal
```

  结果仍是平移 + 旋转 + 等比缩放的相似变换，可以无倾斜地分解回节点字段；后代局部结构保持不变。粘贴前验证整批新子树的深度、节点数和世界有效缩放，任一非法则原子拒绝且剪贴板不变。
- 粘贴保留节点自身的名称、锁定和隐藏字段；锁定或隐藏副本可能继续不可见/不可编辑，但图层树会选中新副本并允许管理元数据。当前编辑容器有效锁定时拒绝粘贴且剪贴板不变。
- 复制页面递归复制场景树和全部 ID，避免不同页面共享节点 ID。
- 删除组选中项时删除整棵子树；锁定节点拒绝删除。
- 图片元素和形状图片填充仍保存现有 `src` 引用。
- `collectFreeformImageSources`、本地 Data URL 物化、远程图片续租和字体收集全部改为递归遍历，并包含隐藏后代。
- 后端草稿 API、SQLite schema 和图片 GC 无需改变；后端已递归扫描草稿 JSON。

## 13. 历史、保存和错误处理

- 文档操作继续使用完整文档快照历史。
- 一次拖动、缩放、旋转、组合、解组、锁定、隐藏、重命名或排序各写入一条历史。
- 展开图层树、切换右侧标签和进入/退出编辑范围不进入历史。
- 草稿保存使用操作开始时的 v3 快照；现有单飞和迟到响应保护继续生效。
- 所有文档快照变化后按第 7.1 节统一校验编辑路径与选区；打开另一份草稿或切换用户时明确回到根编辑范围。
- 非同级组合、跨父级多选、被有效锁定内容的非元数据操作和非法变换不修改文档；权限矩阵允许的锁定、隐藏、重命名除外。UI 使用非阻断提示说明拒绝原因。
- 不新增 HTTP 错误码或服务端配置。

## 15. 性能与兼容性

- 场景派生值按文档快照和活动页 memoize，避免每次指针移动重建整棵树。
- 路径更新只复制祖先链；拖动期间使用 live snapshot，松手后写一条历史。
- 正常目标为单页数百节点；归一化安全上限不是推荐容量。
- 递归函数必须显式传递深度，不能依赖无界调用。
- 继续支持当前系统 Chrome 和 Playwright `channel: 'chrome'`。
- 不操作或关闭用户现有 Chrome；开发预览使用独立端口和独立浏览器上下文。
